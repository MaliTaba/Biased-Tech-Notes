# Biased Tech Notes
A collection of **opinionated engineering notes** from real-world production systems.

These aren’t neutral tutorials or “best practices.” Every decision reflects **trade-offs, constraints, and experience**. Bias is intentional—because real engineering isn’t objective.

Many people, especially outside the tech world, imagine engineering as a purely objective, mathematical discipline. Some think that for any given problem, there is one single, perfect, "engineered" solution that can be calculated.

This is a myth.

Real-world engineering is complex and messy.

# What you’ll find here
Opinions on tech topics.

Some lessons learned from production, not theory.


# How to read this
Disagree freely. 

Be constructive in your comments.

If a note feels wrong, it probably means your constraints differ—and that’s the point.
Take what’s useful, ignore what isn’t, and always apply your own judgment.
